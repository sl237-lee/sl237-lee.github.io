QVC Unified Vendor Data Foundation

Demonstrating how QVC can unify data from TikTok, Meta, and YouTube into a governed Azure data foundation with validation and APIs.

Project Background

This project was inspired by my conversation with Nicholas Kalpakis during the QVC Data Engineer interview. He emphasized the need to build a unified foundation for multiple vendor data sources. I created this demo to illustrate how that could work—combining data contracts, automated validation, and an API layer within an Azure-style design.

Architecture Overview

The design mirrors a typical Azure data pipeline:

ADF → ADLS → Databricks → Synapse → FastAPI

Key Components

  • Data Contracts: YAML-based schemas defining required fields, data types, and rules for each vendor.
  • Data Quality Checks: Automated Python validations for schema mismatches, nulls, negatives, and logical errors.
  • Normalization: Cleans and merges vendor data into a single curated dataset (unified_campaign_performance.csv).
  • API Layer: FastAPI endpoints exposing unified metrics and data-quality summaries.
