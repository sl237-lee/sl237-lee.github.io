QVC Unified Vendor Data Foundation

A demo project showing how QVC can unify data from TikTok, Meta, and YouTube into a governed Azure data foundation with validation and APIs.

Project Background

This project was inspired by my conversation with Nicholas Kalpakis during the QVC Data Engineer interview process. He emphasized the goal of building a unified foundation across multiple vendor data sources. I designed this demo to illustrate how I would approach that challenge—combining data contracts, data quality automation, and an API layer in an Azure-style environment.

Architecture Overview

The design mirrors a typical Azure data pipeline:

ADF → ADLS → Databricks → Synapse → FastAPI

Key Components

  • Data Contracts: YAML-based schemas define expected fields, data types, and validation rules for each vendor (Meta, TikTok, YouTube).
  • Data Quality Checks: Automated Python DQ runner flags schema mismatches, nulls, negatives, or logic errors before transformation.
  • Normalization: Vendor datasets are cleaned and merged into a single curated dataset, unified_campaign_performance.csv.
  • API Layer: FastAPI endpoints expose unified metrics and data quality summaries for analytics or personalization use cases.

Reflection & Impact

This demo highlights how to standardize and operationalize social commerce data in a scalable, governed way. It represents how I’d help QVC:

  • Build a unified vendor foundation for campaign and engagement analytics.
  • Ensure data reliability through contract-based validation and automated quality checks.
  • Expose insights efficiently via APIs and dashboards for downstream teams.

My goal is to bring this same combination of engineering rigor and analytical design to QVC’s social commerce data ecosystem.
